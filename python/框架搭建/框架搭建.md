## 字符串判断

if _ in ["","",""]

- 使用 in 针对列表判断，避免使用 or 拼接复杂条件
- 不使用 int 转换用户输入，避免非数字输入导致错误

## 注意习惯

- 用pass先制定框架代码
- 程序每一步都注释上思维步骤
- 避免代码过长，用多个函数调用解决

	- 将框架代码都可以做成函数或模块
	- 未完成的功能，用TODO，完成后删除

- 定义了函数记得添加注释
